# Experiment Results: Stochastic Edge Weights

## The Problem with Classical Algorithms on Uncertain Weights

Dijkstra makes **greedy, deterministic choices** based on edge weights at query time.
If weights are stochastic (drawn from distributions), the 'cheapest looking' edge
at query time may be much more expensive **on average**.

QI's Metropolis acceptance criterion **inherently models uncertainty** —
accepting suboptimal-looking moves probabilistically mirrors how stochastic
systems actually behave under weight uncertainty.

## Conclusion

**Why QI is better on stochastic problems:**
- Dijkstra commits to the path that looks cheapest using *mean* weights — but high-CV edges
  are often far more expensive than their mean in practice.
- QI naturally explores diverse paths (including 'suboptimal looking' ones on means)
  which happen to be more *robust* when weights fluctuate.
- The Metropolis criterion's probabilistic acceptance is not a bug — it's exactly the
  right behavior for uncertain environments.
